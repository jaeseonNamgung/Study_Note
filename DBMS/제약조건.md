# 제약조건
제약조건이란 데이터베이스의 저장되는 데이터의 정확성을 보장하기 위하여 키를 이용하여 입력되는 데이터에 제한을 주는 것으로 **개체 무결성, 참조 무결성, 도메일 무결성** 등이 있다. 

## 무결성 이란?
데이터의 정확성, 일관성, 유효성이 유지되는 것

- 정확성 : 데이터는 애배하지 않아야 한다.
  - ex) 노란색 O, 빨간색 O, 밝은색 X, 어두운 색X  
- 일관성 : 각 사용자가 일관된 데이터를 볼 수 있도록 해야한다. 
- 유효성 : 데이터가 실제 존재하는 데이터여야 한다. 

1. 개체 무결성 : 모든 테이블이 PK로 선택된 컬럼을 가져야한다. PK로 선택된 컬럼은 고유한 값을 가져야 하며, 빈 NULL 값은 허용하지 않는다. 

2. 참조 무결성 : 두 테이블의 데이터가 항상 일관된 값을 가지도록 유지하는 것이다.

3. 도메인 무결성 : 컬럼의 타입, NULL 값의 허용 등에 대한 사항을 정의하고 올바른 데이터가 입력되었는 지를 확인하는 것이다. 


# 제약조건 종류

- PRIMARY KEY(기본 키)
  - 테이블 행 데이터를 고유하게 식별하기 위한 키이다.
  - 하나의 테이블에 하나의 기본 키 제약만 정의 가능하다. 
  - ※ 기본키 제약 : 고유키 제약 + NOT NULL제약

- UNITUE KEY(고유 키)
  - 테이블 행 데이터를 고유하게 식별하기 위한 고유 키 
  - NULL은 고유 키 제약의 대상이 아니다.

- NOT NULL
  - NULL 값의 입력을 받을 수 없다.(해당 컬럼의 입력 값은 필수)
  - CHECK의 일부분으로 이해할 수 있다.

- CHECK
  - 일력할 수 있는 값의 범위를 제한

- FOREIGN KEY(외래 키)
  - 테이블간의 관계를 정의하기 위해 기본키를 다른 테이블의 외래키로 복사하는 것이다.
