# DB(Database)
- 데이터가 모여있는 기지 
- 통합하여 관리되는 데이터의 **집합체**

# DBMS(Database Management System)
- 데이터를 관리할 수 있는 구체적인 시스템
- 오라클, 마리아DB, MySQL, MS-SQL, Mango,...

> 데이터베이스의 장단점(구글 글 인용)
> - 데이터베이스의 장점
> 1. 데이터 중복 최소화
> 2. 데이터 공유
> 3. 일관성, 무결성, 보안성 유지
> 4. 최신의 데이터 유지
> 5. 데이터의 표준화 가능
> 6. 데이터의 논리적, 물리적 독립성
> 7. 용이한 데이터 접근
> 8. 데이터 저장 공간 절약
> - 데이터베이스의 단점
> 1. 데이터베이스의 전문가 필요
> 2. 많은 비용 부담
> 3. 데이터베이스 백업과 복구가 어려움
> 4. 시스템의 복잡함
> 5. 대용량 디스크로 엑세스가 집중되면 과부하 발생

## 오라클 DBMS 버전
- 계정
  - sys : 모든 권한
  - system : 계정 관리
  - 일반 계정 : 해당 스키마 관리(hr, op, he, scott, ...)

### 스키마 란?
스키마 란 정리가 잘 되어 있는 표들의 묶음, 상태이다.

## RDBMS(관계형 데이터베이스 시스템)
- 테이블끼리 서로 관계를 맺는다.

## SQL(인터프리터 언어), 쿼리문
- 한 줄씩 번역되고 빈번한 수정이 있을 때 효율적이다.
- 전체를 실행하지 않고 부분만 실행하고 싶을 때에도 편리한 언어이다.
- 일괄처리를 할 때에는 컴파일 언어보다 효율이 떨어진다.

## DBMS의 소통 방식

```
-------------------------------------
             사용자
-------------------------------------
    ↕          ↕             ↕ 
고객 관리                 주문관리
응용 프로그램           응용 프로그램

-------------------------------------
```
