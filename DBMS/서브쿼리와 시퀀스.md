# 서브쿼리(Sub + query)
- 두 개 이상의 SQL 문을 하나의 SQL 문으로 연결하여 처리한다.
- 하나의 SQL 문의 결과를 다른 SQL 문에게 전달하기 위해 주로 사용한다. 

## 단일행 서브쿼리 
- 서브쿼리가 단 하나의 행만을 검색하여 메인 쿼리에 반환하는 방식 
- 메인쿼리의 ```WHERE``` 절에서 단 일행 서브쿼리의 결과와 비교할 때는 단일행 비교 연산자를 사용한다. 
- 단일행 비교 연산자 : ```=, !=, >, >=, <, <=```
 
## 다중행 서브쿼리 
- 서브쿼리가 두 개이상의 행을 검색하여 메인쿼리에 반환하는 방식 
- 메인쿼리의 ```WHERE``` 절에서 다중행 서브쿼리의 결과와 비교할 때는 다중행 비교연산자를 사용한다. 
- 다중행 비교 연산자 : ```IN, ALL...```

## CREATE TABLE + 서브쿼리
-   ```CREATE TABLE``` 명령문과 서브쿼리를 결합하여 다른 테이블의 구조와 새로운 테이블에 생성 
- 서브쿼리의 출력 결과를 새로운 테이블의 초기 데이터로 사용할 때 많이 사용한다.
- 서브쿼리의 출력 결과를 값이 없게끔 하면 테이블 구조만 복사된다.
- 서브쿼리 단점 : 데이터 형식만 복사되고 새로 생성한 테이블에 별도 ```PK, FK```등을 다시 정의해야한다.
- 형식 : 

```SQL
CREATE TABLE [새로운 테이블](속성1, 속성2...)AS[서브쿼리문];
```

## INSERT + 서브쿼리
- ```INSERT``` 문에서 ```VALUES``` 절 대신 서브쿼리를 작성하여 결과를 한 번에 삽입이 가능하다. 
- 형식 : 

```SQL
INSERT INTO [테이블 이름](속성1, 속성2)[서브쿼리문];
```
- 기본 키가 설정된 속성에 중복된 삽입하는 경우에는 데이터가 입력되지 않는다.

## UPDATE + 서브쿼리
- 형식 : 
```SQL
UPDATE [테이블 이름] SET(속성1, 속성2...) = [서브쿼리문] WHERE 조건식;
```
## DELETE + 서브쿼리
- 형식 : 
```SQL
DELETE FROM [테이블 명] WHERE(속성1, 속성2...) = [서브쿼리문];
```

## 시퀀스(SEQUENCE)
- 자동으로 증가하는 순번을 만드는 데이터베이스 객체로 기본 키 속성에 중복 값을 방지한다. 
- 주로 글이 하나 추가될 때마다 글 번호(PK)가 생겨나는 게시판 테이블에 사용한다. 
- 시퀀스 객체를 사용하면 데이터가 추가될 때마다 자동으로 글 번호를 1씩 증가시킨다.
- 시퀀스 기본 형식 : 
```SQL
CREATE SEQUENCE[시퀀스 이름]
START WITH[시작 번호]
INCREMENT BY [증가 값]
MINVALUE[최소값]
MAXVALUE[최대값];
```
** CYCLE 또는 NOCYCLE : CYCLE 설정 시 최댓값에 도달하면 최솟값 부터 다시 시작한다. 
- 시퀀스 삭제 : 
```SQL
DROP SEQUENCE[시퀀스 이름];
```
