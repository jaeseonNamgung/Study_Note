# 1. 데이터 모델링 이란? 
- 현실 세계에서 사용되는 작업이나 사물들을 Oracle 같은 DBMS에 어떻게 구현할지 결정하는 단계
- 즉, 현실 세계를 데이터베이스에 표현하고 변경하는 중간 과정이다.
- 현실 세계에 존재하는 개체에 대한 특성과 특징을 파악하여 구체화시킨다. 
- 예) 인터넷 쇼핑몰 데이터베이스를 구축할 경우 사용할 개체와 속성들은 아래와 같다.
  - 인터넷 회원(개체)의 속성
    - ID, 이름, 연락처, 집, 주소, 이메일 주소, 결제 수단 정보 등
  - 상품(개체)의 속성
    - 상품명, 상품번호, 가격, 브랜드, 제조일자, 남은 수량, 배송 방법 등

# 2. 개체 관계 모델(Entity-Relationship Model)
## E-R 모델의 특징
- 데이터 모델링의 개념적 설계 단계에서 가장 많이 사용되는 모델이다.
- 개체(Entity)와 관계(Relationship)를 이용하여 현실 세계를 표현한다. 
- 개체(Entity)는 현실 세계에 존재하는 학생, 교수 등을 의미한다.
- 관계(Relationship)는 개체들 사이의 관계를 표현하는 것으로 대여, 수강 등에 해당한다.
- E-R 다이어그램은 그래프 형태로 표시한다.

## E-R 모델 관계 타입의 유형 
- 1:1 관계(일 대 일)
  - 개체 집합 A의 각 원소가 개체 집합 B의 원소 1개와 대응
  
![image](https://user-images.githubusercontent.com/84066249/137571433-132d8dcb-5a4f-47c6-9d96-01975eb2534d.png)

(한 명의 교수는 한 과목만 강의하고, 한 개의 과목은 한 명의 교수에 의해 강의 된다.)
- 1:N 관계(일 대 다)
  - 개체 집합 A의 각 원소는 개체 집합 B의 원소 여러 개와 대응할 수 있고, 개체 집합 B의 각 원소는 개체 집합 A의 원소 1개와 대응
  
![image](https://user-images.githubusercontent.com/84066249/137571429-ee1c41fa-199b-4e98-add7-61c4b79747b4.png)

(한 학과에는 여러 명의 학생이 있을 수 있고 한 학생은 한 개의 학과에 소속된다.)

- N:M 관계(다 대 다)
  - 개체 집합 A의 각 원소는 개체 집합 B의 원소 여러 개와 대응할 수 있고, 개체 집합 B의 각 원소는 집합 A의 원소 여러 개와 대응할 수 있다. 
  
![image](https://user-images.githubusercontent.com/84066249/137571425-d043aeeb-90b2-45c0-8d31-f5254f95972c.png)

(한 명의 학생은 여러 과목을 수강할 수 있고, 한 과목은 여러 명의 학생에 의해 수강될 수 있다.)
