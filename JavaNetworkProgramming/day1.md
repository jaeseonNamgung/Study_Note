# Network란 
- 하나의 Device(컴퓨터)로 부터 다른 Device로 데이터르 옮기는 것
- Network Programming : Network로 전달된 정보를 저장, 분석 처리하는 것

## 통신의 3대 요소
- 서버(Server)
- 클라이언트(Client)
- Network : 서버와 클라이언트를 연결해주는 것 

# IP(Internet Protocol)
- 네트워크상에서 컴퓨터는 다른 컴퓨터와 구별 될 수 있도록 고유번호를 가지게 되는데, 이것은 인터넷에 접속할 때 컴퓨터 각각에 부여 받는 **주소**이다.
- 각 나라별 나라의 공인기관에서 IP를 할당하고 관리한다. (우리나라는 한국인터넷 진흥원에서 이를 담당)
- IP는 3자리 숫자가 4마디씩 표기( 예) 000.000.000.000.)
- IP는 목적지로 데이터를 전송하기 위해서 중간에 어떤 경로를 거쳐갈 것인지를 정하는 프로토콜이다. 

# 프로토콜(Protocol)
프로토콜은 약속, 규약, 협약 이란 뜻으로 상호간 원할한 교류, 소통, 통신을 위해 서로 동일한 어떤 규칙 혹은 약속을 정한 것이다. 

# 전송계층
**TCP** : IP에서 알려준 경로정보를 바탕으로 데이터의 실제 송수신을 신뢰성있는 데이터의 전송을 담당한다. TCP가 데이터를 보낼 때 기반이 되는 프로토콜 IP이다.
(예 :  A에서 B로 데이터를 전송할 때 A가 B에게 전달을 잘 했는지 안 됐는지 알려주고 만약 안 되어 있다면 데이터를 다시 전송한다.)
**TCP는 신뢰성이 높고 UDP보다는 속도가 떨어진다.**

**UDP** : IP에서 알려준 경로정보를 바탕으로 데이터의 실제 송수신을 담당. 비연결 지향성 프로토콜이다.(UDP는 데이터가 전송이 됐는지 안 됐는지 알려주지 않는다.)
**TCP에 비해 신뢰성은 떨어지지만 속도는 TCP보다 빠르다.**

# 소켓(Socket) 
소켓이란 프로그램상에 동작하는 프로그램 간 통신의 종착점이다. 즉 프로그램과 프로그램 사이에 통신할 수 있도록 연결 해주는 역할이 소켓이다.
