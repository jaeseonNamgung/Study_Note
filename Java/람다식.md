# 람다식이란?
람다식은 Java SE8에서 새로 도입되었고 자바 8에서 가장 핵심적인 기능이다. 람다식은 함수를 만들때 클래스를 만들지 않고 바로 람다식을 만들어 함수처럼 사용할 수 있다. 

> **메서드 생성**  
 int add(int x, int y)
 {
   return x + y;
 }
 
 
> **람다식 생성**  
  (x,y) -> x + y 


위에 표를 보면 두 코드는 같은 기능을 수행하는 코드이다. 하지만 보면 바로 알 수 있듯이 코드 길이에 차이가 크다. 자바에서는 원래 단독으로 함수를 사용할 수 없다. 함수를 사용하기 위해서는 클리스를 만들어 클래스안에서 메서드 실행이 가능하다. 하지만 람다식을 이용하게 되면 자바에서도 함수를 단독으로 사용할 수 있다. 즉 더 간결하게 코드를 작성할 수 있다. 

# 람다식 사용방법

> (데이터 타입 매변수, ...) -> {실행 문,...}

1. (int x) -> {System.out.println(x);} : 기본형 람다식
2. (x) -> {System.out.println(x);} : 매개 변수의 타입을 추론할 수 있는 경우(데이터 타입을 명시하지 않아도 문제가되지 않는 경우)에는 타입을 생략 가능하다.
3. x -> System.out.println(x) : 매개 변수나 실행문이 하나라면 소괄호 ()와 중괄호 {}를 생략 가능하다. 이 때 세미콜론 ;은 붙이지 않는다.
4. () -> {System.out.println(x);} : 매개 변수가 없을 경우에는 빈 소괄호 ()를 사용한다. 괄호는 생략할 수 없다.
5. (x,y) -> {return x + y;} : 함수식이 하나의 return 문으로만 이루어진 경우에는 줄괄호 {}를 생략할 수 없다.
6. (x,y) -> x + y : return 문 대신 표현식만 사용 가능하며, 이 때 리턴 값은 표현식의 결과 값이 된다. 이 때 세미콜론 ;은 붙이지 않는다. 


# 예제


``` java
interface LamdaInterface {
	public int calculate(int x, int y); // 연산 결과를 리턴하는 추상 메서드
}

class Person {

	public void printInfo(LamdaInterface li) {
		int sum = li.calculate(5, 10);
		System.out.println("sum : " + sum);
	}
}

public class LamdaTest {

	public static void main(String[] args) {
		// TODO Auto-generated method stub

		Person person = new Person();

		person.printInfo(new LamdaInterface() { // 람다식을 사용하지 않을 경우
			public int calculate(int x, int y) {
				return x + y;
			}
		});

		person.printInfo((x, y) -> {
			return x - y;
		}); // 람다식을 사용했을 경우

	}

}
```
