# 레퍼런스(reference)란?
레퍼런스(reference)란 참조라는 뜻을 가진다. 즉 레퍼런스는 그 메모리에 값을 바로 사용하는 게 아니라 다른 메모리에 값을 참조해서 다른 메모리에 값을 사용한다.

- **레퍼런스**
  - 클래스 타입
  - 사용자 정의 타입
- **기본 형**
  - byte
  - int
  - long
  - float
  - double
  - boolean 등

위 사진에서 변수 a는 int형이기 때문에 변수 a는 일반 변수이다. 그렇기 때문에 변수 a는 Stack 메모리에 값을 바로 사용 가능하다. 
Car은 클래스 타입이기 때문에 car 변수는 레퍼런스 변수이다. 위 사진에서 보면 알 수 있듯이 car 변수는 Stack 메모리에 바로 값을 사용하는 게 아니라 Heap 영역에 생성된 클래스 변수 혹은 메서드를 참조해서 사용한다.  

## 레퍼런스 변수 사용법 
```java
레퍼런스 변수.변수 명

     or
     
레퍼런스 변수.메서드 명

```
레퍼런스 변수가 Heap에 있는 변수나 메서드를 참조하기 위해서는 레퍼런스 변수뒤에 .을 적고 변수나 메서드 이름을 적으면 된다.

# 인스턴스 변수, 클래스 변수, 지역변수

- 지역(로컬 변수)
```java
클래스가 아닌 블록 영역{
                                     
    로컬 변수
    
}

```

```java
클래스 영역{

멤버 변수 -> 1. 인스턴스 변수
             2. 클래스 변수
}
```


```java
class a{

int iv;  // 인스턴스 변수

static int cv;  // 클래스 변수

void method(){
    int mv;  // 지역 변수
}

}

```

- 인스턴스 변수: 클래스 영역에 정의되고 static 키워드가 아닌 일반 멤버 변수를 인스턴스 변수라 한다.
                   인스턴스 변수는 객체가 생성될 때 각 객체별로 초기화되며, 선언할 때 변수를 초기화하지 않으면 컴파일러가 자동으로 기본 값으로 초기화 시켜준다.
                   
- 클래스(static) 변수: 클래스 영역에 정의되고 변수 이름 앞에 static 키워드를 사용한다. 클래스 변수는 값을 초기화하지 않으면 컴파일러가 자동으로 기본 값으로 초기화 시켜준다.
                      클래스 변수는 클래스를 로드할 때 Class Area의 Static 영역에 바로 초기화되기 때문에 해당 클래스 타입으로 생성된 모든 객체에서 공유된다. Static 멤버 변수(클래스 멤버 변수)는 다른 클래스에서 객체를 생성하지 않고도 해당 클래스 이름으로 바로 참조가 가능하다. (이유는 공유하고 있기 때문)

- 로컬 변수(지역변수): 로컬 변수는 메서드나 초기화 블록에서 선언한 변수이다. 로컬 변수는 개발자가 변수를 초기화하지 않으면 컴파일 시 오류가 난다. 로컬 변수는 정의된 메서드나 초기화 블록 안에서만 사용 가능하다. 로컬 변수는 메서드나 초기화 블록이 실행되면 Stack 영역에 초기화되었다가 메서드나 최기화 블록이 실행이 종료되면 로컬 변수는 자동으로 메모리에서 소멸된다.
