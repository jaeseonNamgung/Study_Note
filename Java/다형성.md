자바에서 다향성이란 하나의 레퍼런스 변수가 여러 타입의 객체를 참조하는 것을 의미한다.

```
             Person1        
        ↗      ↑        ↖
 Salaryman1 Student1 President1

```

위에 그림처럼 하나의 부모 클래스는 여러 개의 자식 클래스를 상속할 수있다.
즉 자식 클래스는 부모 클래스에 있는 기능을 사용할 수 있다. 그러나 다형성을 이용하면 부모 클래스에서 자식 클래스에 기능을 사용할 수 있다.


# 업 캐스팅(UpCasting), 다운 캐스팅(DownCasting)

- 업 캐스팅(UpCasting)

```java
Person1 person = new Salaryman1();
Person1 person = new Student1();
Person1 person = new President1();

```
상단의 예처럼 부모 클래스의 레퍼런스 변수가 자식 클래스를 참조해서 자식 클래스 객체의 레퍼런스 값의 타입이 부모 클래스 타입으로 변환된다. 이를 업 캐스팅(UpCasting)이라고 한다.
즉 자식클래스의 타입이 부모 클래스 타입으로 형 변환 되는 것이다.
반대로 자식 클래스의 레퍼런스 변수가 부모 클래스를 참조해서 부모 크래스의 레퍼런스 값의 타입이 자식 클래스 타입으로 변환 되는 것을 다운 캐스팅(Down Casting)이라고 한다.
즉 부모 클래스 타입이 자식 클래스 타입으로 형 변환 되는 것이다.
- 다운 캐스팅(Down Casting)
```java
Person1 person = new Person1();
President1 president1 = person(); // 컴파일 오류
President1 president1 = (President1)person;  // 런타임 오류
```
자바에서는 자식 클래스 타입의 레퍼런스 변수가 부모 클래스의 레퍼런스 값을 참조하는 것을 혀용하지 않는다. 왜냐하면 부모 클래스에서 상속 받은 자식 클래스들은 부모클래스의 기능을 사용할 수 있을 뿐 아니라 자식 클래스에서 정의한 기능도 사용할 수 있다. 그렇기 때문에 자식 클래스는 부모 클래스 보다 더 많은 기능들이 구현 되어 있을 수 있다. 그렇기 때문에 자식 클래스로 호출하는 변수나 메서드가 Heap영역에 생성되어 있는 부모 클래스 객체에 없는 것일 수 있다. 그러나 아래와 같이 사용할 교ㅕㅇ우
