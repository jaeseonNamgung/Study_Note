# 추상클래스 
- 추상클래스는 body가 없는 클래스이다. 즉 정의만 해놓고 구현은 하지 않은 메서드가 있는 클래스이다. 
- 추상클래스는  ```abstract``` 키워드를 사용한다. (클래스명 앞과 메서드명 앞에 정의해 줘야 한다.)
- 추상메서드를 한 개 이상 가지고 있는 클래스는 ```abstract```키워드를 통해 반드시 추상클래스화 되어 있어야 한다.
- 부모인 추상클래스는 미완성된 추상메서드를 가지고 있으므로 이 미완성의 메서드를 자식이 받아서 반드시 자식 기능에 맞춰 재정의 해야한다.
- 추상클래스는 인스턴스를 직접 가질 수 없다.(단독으로 객체와 불가능)

# 인터페이스 
- 추상클래스와 유사하지만, 서비스의 요청에 따른 중개자 역할을 하는 클래스이다.
- 인터페이스에는 ```상수```와 ```추상메서드``` 이외에는 아무것도 정의할 수 없다.
- 인터페이스에서는 ```final```와 ```abstruct```를 따로 지정하지 않아도 된다.(인터페이스는 상수와 추상메서드만 정의할 수 있기 때문)
- 상수는 변수명을 대문자로 작성하는 것을 권장한다.
- ```implement``` 키워드를 통해 인터페이스를 구현한다.
- 인터페이스도 직접적으로 객체화할 수 없다.
- 인터페이스는 다중 상속이 가능하다. (부모가 두 개 이상 가능)
- 인터페이스들 간에는 ```extends```를 통해 상속이 가능하며 **인터페이스만 유일하게 다중 상속이 가능하다.**
