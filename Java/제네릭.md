# 제네릭이란?
제네릭이란 클래스 내부에 데이터 타입을 외부에서 정의하는 것을 말한다. 제네릭은 **<T>** 꺽세를 사용하며 <>안에 내용을 넣으면된다. 또 한 제네릭은 래퍼(wrapper) 클래스를 사용해야한다.(기본 데이터 타입 사용 불가능)
  
  
### 제네릭 사용법
  
 
```java
  
class GenericClass<T> { // 제네릭 사용

}

public class GeneriTest {

	public static void main(String[] args) {

		GenericClass<Integer> g = new GenericClass<Integer>(); // 메인 메서드에서 제네릭 데이터 타입 지정

		GenericClass<String> g1 = new GenericClass<String>(); // 메인 메서드에서 제네릭 데이터 타입 지정
	}

}

 
```

GenericClass에서 <T>안에 T는 아무 값도 없는 상태이다. 즉 어떠한 데이터도 아니다. T는 메인 메서드에서 정의해준 데이터 타입에서 데이터 형이 정해진다.  
  
  
# 제네릭을 사용하는 이유
	
1. 형 변환
	
**프로그램에서 형 변환을 자주 해 줄 경우 성능적으로 좋지않다.**


- 제네릭을 사용하지 않은 경우
	
```java
class ObjectClass { 

	private Object object;

	public Object getObject() {
		return object;
	}

	public void setObject(Object object) // String -> Object 형 변환
	{
		this.object = object;
	}

}

public class GeneriTest {

	public static void main(String[] args) {

		ObjectClass o = new ObjectClass();

		o.setObject("NonGeneric");

		String str = (String) o.getObject(); // Object -> String 형 변환

		System.out.println(str);

	}

}
	
```
위에 클래스는 단순히 하나의 변수를 setObject 메서드에서 변수를 초기화한 후 getObject 메서드에서 값을 반환하는 코드이고 메인 메서드에서 반환 받은 값을 출력해주는 코드이다. 
여기서 주목해야될 점은 클래스 내부에서 Object 객체로 데이터 형을 정해준 것이다. 모든 클래스는 Object 객체를 상속 받는다. 즉 Object 타입으로 데이터 타입을 정의할 경우 어떤 타입이든 다 받을 수 있다. 하지만 Object 타입으로 받으면 항상 데이터 간의 형 변환이 필요하다. 위에 코드에서 확인해 보면 메인 메서드에서 getObject() 메서드로 인자를 넘겨 줄경우 Object 타입으로 형 변환이 이루어지고  getObject() 메서드에서 값을 반환 받을경우 Object 타입에서 String 타입으로 형 변환을 해줘야한다. 위에서 언급했듯이 형 변환이 많으면 성능적으로 좋지 않기 때문에 위 코드느 좋지 않은 코드이다.

	
- 제네릭을 사용 할 경우 

```java
	
class GenericClass<T> { // 제네릭 사용

	private T generic;

	public T getGeneric() {
		return generic;
	}

	public void setGeneric(T generic) {
		this.generic = generic;
	}

}

public class GeneriTest {

	public static void main(String[] args) {

		GenericClass<String> o = new GenericClass<String>();

		o.setGeneric("Generic");

		String str = o.getGeneric();

		System.out.println(str);

	}

}
	
```
제네릭을 사용 할 경우 객체 생성시 데이터 타입을 지정해 주기 때문에 형 변환이 필요없게 되면서 성능적으로 더 좋아진 걸 알 수 있다. 

2. 같은 기능을 하는 다른 데이터 타입
	
- 제네릭을 사용하지 않은 경우

```java
class StringClass {

	private String str;

	public String getStr() {
		return str;
	}

	public void setStr(String str) {
		this.str = str;
	}

}

class IntegerClass {

	private Integer inte;

	public Integer getInte() {
		return inte;
	}

	public void setInte(Integer inte) {
		this.inte = inte;
	}

}

public class GeneriTest {

	public static void main(String[] args) {

		StringClass sc = new StringClass();
		IntegerClass ic = new IntegerClass();
		sc.setStr("str");
		ic.setInte(1234);

		System.out.println(sc.getStr());
		System.out.println(ic.getInte());

	}
}
	
	
```
	
위에 코드를 보면 같은 기능을 하는 클래스를 확인할 수 있다. 즉 데이터 타입이 다르기 때문에 기능이 같은 클래스를 두 개 정의한 것이다.

- 제네릭을 사용할 경우
	
```java
class GenericClass<T> {

	private T generic;

	public T getGeneric() {
		return generic;
	}

	public void setGeneric(T generic) {
		this.generic = generic;
	}

}

public class GeneriTest {

	public static void main(String[] args) {

		GenericClass<String> str = new GenericClass<String>();
		GenericClass<Integer> inte = new GenericClass<Integer>();

		str.setGeneric("str");
		inte.setGeneric(1234);

		System.out.println(str.getGeneric());
		System.out.println(inte.getGeneric());

	}
}
	
```
	
제네릭을 사용할 경우 하나에 클래스에 여러 데이터 타입을 사용할 수 있기 때문에 코드 길이가 짧아 지는걸 알 수 있다.
	

# 복수의 제네릭

```java
class GenericClass<T, S> {

	T str;
	S inte;

	public GenericClass(T str, S inte) {
		this.str = str;
		this.inte = inte;
	}

	public void getInfo() {
		System.out.println(str + "," + inte);
	}

}

public class GeneriTest {

	public static void main(String[] args) {

		GenericClass<String, Integer> g = new GenericClass<String, Integer>("str", 1234);

		g.getInfo();

	}
}	
```
위에 코드 처럼 제네릭은 여러 개 사용이 가능하다. 객체를 생성할 때 제네릭에 데이터 타입을 정의한 순서대로 데이터 타입이 결정된다.
