# 생성자
- 객체가 생성될 때 메모리 할당을 위해 호출되는 메서드 개념의 영역 (원하든 원하지 않든 생성자는 무조건 한 번 실행된다.)
- 생성자는 객체당 한 번만 호출할 수 있다.(만약 값을 변경하지 못하게 작성하고 싶다면 생성자를 통해 구현하면 좋다.)
- 생성자는 반환형이 없다.
- 클래스 이름과 대소문자가 완전히 동일해야 한다.
- 생성자가 오버로딩 되어 있을때 기본 생성자를 만들어두지 않으면 기본 생성자를 자동으로 만들어주지 않는다. 

# static 
- 일반적인 메서드에서는 static 변수를 언제든지 가져다 사용할 수 있다.
- static 메서드에서는 일반 멤버변수를 참조해서 쓸 수 있다.
- static 메서드에서는 static 변수를 선언하거나 생성할 수 없다.
- 해당 클래스를 통한 객체가 아무리 많이 생성되어도 메모리에 오직 한 개만 만들어지게 하기 위한 키워드이다.
- static 변수는 클래스명. 을 통해서 직접적으로 접근이 가능하다.
- static 변수는 객체 생성이 원할 때 언제든 사용 가능하다.
- static은 메모리에서 제일 먼저 호출된다. 그렇기 때문에 static을 너무 많이 사용하면 메모리에서 static 메모리가 제일 먼저 실행되기 때문에 다른 메모리에 접근하는 속도가 느려져서 static을 많이 사용하지 않는 걸 권장한다.
