- 자바 스크립트에서는 문자열 비교는 사전 순(유니코드 순)으로 문자열을 비교한다.
  - 두 문자열의 첫 글자를 비교한다.
  - 앞 글자 부터 순서대로 비교해 앞 글자가 사전 순(유니코드 순)이 빠르면 뒷 글자의 사전 순(유니코드 순)과 상관없이 비교를 종료한다. 
  - 두 문자열이 같다면 다음 문자열을 비교한다.
  - 두 문자열의 사전 순(유니코드 순)이 같고 길이가 다르다면 길이가 긴 문자열이 크다고 판단한다.
  - 소문자와 대문자를 비교할 경우 소문자가 더 크다
- 문자열 앞에 +기호를 사용할 경우 숫자형으로 변환된다.
- 2 > 3처럼 비교 연산자는 불린형으로 값을 반환한다. 
- 비교하려는 값이 다르면 자바스크립트는 이 값들을 숫자형으로 변환한다.
- 일치 연산자(scrict equality operator) ```===```를 사용하면 형 변환 없이 값을 비교할 수 있다. (비교할 두 값이 서로 형이 다르다면 즉시 false를 반환한다.)
- null과 undefined 비교
  - null === undefined의 비교 값은 false이다.
  - null == undefined의 비교 값은 true이다.
  - null > 0의 비교 값은 false이다.
  - null == 0의 비교 값은 false이다.
    - ```==```은 형 변환을 하지 않는다.
  - null >= 0의 비교 값은 true이다.
    - ```>=``` 기타 연산자의 동작 방식의 따라 null이 숫자형으로 변환된다.
  - undefined는 다른 값과 비교할 수 없다 즉 false를 반환한다.
