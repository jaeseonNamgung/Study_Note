# 제약 조건(Constraint)
제약 조건이란 뷰가 레이아웃 안의 다른 요소와 어떻게 연결되는지 알려주는 것으로 뷰의 연결점(Anchor Point)과 대상(Target)을 연결한다.  
![image](https://user-images.githubusercontent.com/84066249/132290425-af9fc9d6-a20c-46f2-91ee-af6725829ab8.png)
그림에서 버튼에 입장에서 버튼을 감싸고 있는 레이아웃은 부모 레이아웃(Parent Layout)이라고 부른다. 그리고 버튼은 상하좌우 각각 연결점을 가지고 있다. 만약 버튼에 연결점이 부모 레이아웃에 왼쪽 벽면에 연결하면 부모 레이아웃은 버튼에 타킷(Target)이 된다. 이렇게 연결점과 타깃이 연결되면 연결 선이 만들어지고 파랗게 표시된다.  또 한 버튼이 어디에 위치하는지 알기 위해서는 적어도 왼쪽과 위쪽에 연결되어 있어야 한다. 즉 부모 레이아웃의 왼쪽과 위쪽에 연결되면 제약 조건이 완성된다.

- 타킷이 될 수 있는 것들
 1. 같은 부모 레이아웃 안에 들어 있는 다른 뷰의 연결점
 2. 부모 레이아웃의 연결점
 3. 가이드라인(Guideline)

- 연결점이 될 수 있는 것들
 1. 위쪽(Top), 아래쪽(Bottom), 왼쪽(Left), 오른쪽(Right)  (Left, Right는 Start, End로 표현되기도 한다.)
 2. 가로축의 가운데(CenterX), 세로축의 가운데(CenterY)
 3. 베이스라인(Beseline) ->  텍스트를 보여주는 뷰인 경우에만 적용된다.
 

**마진(Margin): 마진은 연결점과 타깃과의 거리를 나타낸다.**


![image](https://user-images.githubusercontent.com/84066249/132321911-6ebd0fad-2c81-4b22-9f65-4f77b32527af.png) : 부모 여유공간을 채운다. -> layout_width의 값은 match_constraint나 0dp가 된다.

![image](https://user-images.githubusercontent.com/84066249/132322260-f990c729-09d6-4c57-84df-e5820d94c201.png) : 뷰의 내용물을 채운다. -> layout_width의 값은 wrap_content로 설정된다.

# 가이드라인
가이드라인(Guideline)은 여러 개의 뷰를 일정한 기준 선에 정렬할 때 사용한다. 가이드 라인은 크기가 0 이고 보이지 않기 때문에 레이아웃에 없는 걸로 간주된다.


![image](https://user-images.githubusercontent.com/84066249/132324455-93c78f32-62d6-4314-ad09-bf211cc3072c.png)

- Add Vertical Guideline : 세로 가이드 라인이 추가된다.
- Add Horizontal Guideline : 가로 가이드 라인이 추가된다.


# XML원본에 추가된 속성 

```xml
<?xml version="1.0" encoding="utf-8"?>
```
가장 위쪽에 있는 줄로 XML 파일에 일반적으로 추가하는 정보이며, 이 파일이 XML 형식으로 된것임을 알려준다. 

```xml
<androidx.constraintlayout.widget.ConstraintLayout 
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent">
```
그 다음 입력되어 있는 코드가 화면 전체를 감싸고 있는 레이아웃이다. Component Tree 창의 계층에서 가장 위쪽에 있고 이를 최상위 레이아웃이라고 부른다. 여기에서는 ConstraintLayout은 이 태그의 이름으로 사용되고 그 앞에는 패키지 이름(androidx.constraintlayout.wid-get)이 사용된다. 만약 레이아운이 안드로이드 기본 API에 포함되어 있다면 그 위젯이나 레이아웃의 이름만 입력하면 된다. 하지만 외부 라이브러리에서 불러온 것이라면 패키지 이름까지 같이 입력해야 된다. (ConstraintLayout은 안드로이드 SDK에 나중에 추가되면서 외부라이브러리로 분류되어 있다) 
그 다음 태그의 속성을 살펴보면 xmlns:로 시작하는 속성들이 있다. 그 중에서 xmlns:android는 xml 레이아웃 파일이라면 한 번쯤 사용하고 한 번만 사용되는 속성이다. xmlns 뒤에 오는 android라는 이름이 나머지 속성의 접두어(Prefix)로 사용된다. 

**접두어 의미**
- xmlns:android : 안드로이드 기본 SDK에 포함되어 있는 속성을 사용한다.
- xmlns:app : 프로젝트에서 사용하는 외부 라이브러리에 포함되어 있는 속성을 사용한다. (app라는 단어는 다른 것으로 변경 가능하다.)
- xmlns:tools : 안드로이드 스튜디오의 디자이너 도구 등에서 화면에 보여줄 때 사용한다. 이 속성은 앱이 실행될 때는 적용되지 않고 안드로이드 스튜디오에서만 적용된다.

### android:id
android:id의 id 속성은 뷰를 구분하는 구분자 역활을 한다. (여러 개의 뷰를 사용했을 때 구분하기 위해서 사용한다.)

**사용 이유**
1. XML 레이아웃 파일 안에서 뷰룰 구분할 때
2. XML 레이아웃 파일에서 정의한 뷰를 자바 소스 파일에서 찾을 때

### XML 속성 이름 규칙

```xml
layout_constraint[소스 뷰의 연결점]_[타깃 뷰의 연결점]="[타깃 뷰의 id]"
```
### id 속성 값 형식
```xml
@+id/아이디 값
```

## 가이드라인 태그 

```xml
<androidx.constraintlayout.widget.Guideline
        android:id="@+id/guideline4"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:orientation="vertical"
        app:layout_constraintGuide_begin="100dp" />
```
가이드라인는  androidx.constraintlayout.widget을 사용하기 때문에 외부라이브러리를 사용하고 있다는 것을 알 수 있다. 태그에는 필수 속성인 layout_width와 layout_height 속성과 값이 들어 있다. 그리고 id값으로 @+id/Guideline이 설정되어 있다. 그 다음 속성에는 android:orientation이 있는데 가이드라인은 가로 또는 세로 방향중에서 어는 방향인지 지정해야 하므로 Guideline에서 필수 속성이다. 그 아래에는 layout_constraintGuide_begin 속성은 부모 레이아웃의 벽면에서 얼마나 떨어뜨려 배치할지 지정하는 속성이다. 

** 벽면에서 얼마나 떨어뜨릴 건지 지정하는 세가지 속성
- layout_constraintGuide_begin : 세로 방향인 경우 왼쪽부터, 가로 방향인 경우 위쪽부터의 거리 지정
- layout_constraintGuide_end :  세로 방향인 경우 오른쪽부터, 가로 방향인 경우 아래쪾부터의 거리 지정
- layout_constraintGuide_percent : layout_constraintGuide_begin 속성 대신 지정하되 % 단위로 거리 지정

** 제약 조건을 설정한느 외부 라이브러리의 속성이므로 속성 이름 앞에 모두 app:접두어가 붙어 있어야한다. 

# 크기를 표시하는 단위와 마진

단위 |단위 표현|설명
|:------|:-------:|:-------:|
px|픽셀|화면 픽셀의 수
dp 또는 dip|밀도 독립적 픽셀|160dpi 화면을 기준으로 한 픽셀 예) 1인치 당 160개의 점이 있는 디스플레이 화면에서 1dp는 1px와 같다. 1인치 당 320개의 점이 있는 디스플레이 화면에서 1dp는 2px와 같다.(해상도 별로 일일이 크기를 다시 지정하지 않아도 된다.)
sp 또는 sip| 축적 독립적 픽셀|텍스트 크기를 지정할 때 사용하는 단위. 가변 글꼴을 기준으로 한 픽셀로 dp와 유사하나 글꼴의 설정에 따라 1sp당 픽셀수가 달라진다.(글꼴을 기준으로 한 텍스트 크기를 나타내므로 뷰의 크기를 지정하는데는 사용되지 않는다., 글자를 표시하는 텍스트 뷰나 버튼의 텍스트 크기는 sp 단위 사용을 권장한다.)
in    |인치|1인치로 된 물리적 길이 
mm    |밀리미터|1밀리미터로 된 물리적 길이
em    |텍스트 크기|글꼴과 상관없이 동일한 텍스트 크기 표시
