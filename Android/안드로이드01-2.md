# 뷰(View)와 뷰 그룹(View Group)
뷰(View): 뷰는 레이아웃, 위젯이라고도 부르며 화면을 구성하는 인터페이스의 구성 요소이다. 
즉 뷰는 안드로이드 화면을 구성요소이며 뷰는 화면에 보이는 것들이다. 

뷰 그룹(View Group): 뷰 그룹은 뷰를 여러 개 포함하고 있는 것을 뷰 그룹이라고 한다. 뷰는 여러 개의 뷰를 담고 있는 그릇과 같은 형태이기 때문에 뷰와 뷰 그룹은 다르다. 하지만 또 다른 뷰 그룹을 뷰 그룹안에 넣고 다른 뷰와 동일하게 만들수 있다. 여러 개의 뷰를 담고 있는 뷰 그룹 역시 또 다른 하나의 뷰라고 했을 때 또 다른 뷰 그룹이 이 뷰 그룹을 뷰처럼 사용할 수 있다. 이러한 뷰와 뷰 그룹의 관계는 **컴포너지 패턴(Composite pattern)** 을 사용하여 만들어진다.


# 위젯과 레이아웃

뷰와 뷰 그룹의 역활을 구분하기 위해 뷰의 종류를 위젯과 레이아웃으로 나뉜다. 

**위젯:** 일반적인 컨트롤의 역활을 하는 것을 위젯(Widget)이라고 부른다.  **화면 구성을 하는 것을 뷰, 사용자 눈에 보이는 컨트롤 역활을 하는 것을 위젯이댜.**

**레이아웃:** 뷰 그룹 중에서 내부에 뷰들을 포함하고 있으면서 그것들을 배치하는 역활을 하는 것을 레이아웃(Layout)이라고 부른다. **뷰를 담고 있는 것을 뷰 그룹, 뷰 그룹 안의 뷰를 배치하는 것을 레이아웃이다.** 

# 뷰의 크기 속성
뷰는 화면의 일정 영역을 차지하기 떄문에 모든 뷰는 반드시 크기 속성을 가지고 있어야 한다. 먼저 XML 원본을 살펴보면 꺽쇠 < ... </>로 이루어진 형태를 볼 수 있다. 이 형태는 태그를 의미하며 꺽쇠안에 글자가 들어간 형태이다. 태그는 시작 태그 끝 태그로 이루어 져있고 끝 태그는 /로 나태낸다.  시작 태그 안에는 공백을 구분으로 하는 속성이 들어가 있다. 
끝 태그는 /> or  </>로 나태낸다. 시작 태그와 끝 태그는 따로 사용이 가능하다.


``` xml
<시작 태그
    속성1="속성값1"
    속성2="속성값2"
    ...
</끝 태그>


<시작 태그
    속성1="속성값1"
    속성2="속성값2"
    ...
끝 태그/>

```


태그 안에 들어간 속성들은 모두 공백이나 줄 바꿈으로 구별된다. 그리고 대부분 android: 이라는 단어가 앞에 붙어 있는데 android: 은 안드로드의 기본 API에서 정의한 속성이다. 
                                                                                                                                                                                                                                                                                                                                                                     ## 뷰가 가질 수 있는 필수 속성 
                                                                                                                                                                                    - layout_width: 가로 크기
                                                                                                                                                                                    - lauout_height: 세로 크기 
                                                                                                                                                                                   이 두 속성은 뷰가 가질 수 있는 필수 속성이다. 그렇기 때문에 XML 레이아웃에 이 두 가지 속성 중 하나라도 없다면 오류가 발생한다. 
                                                                                                                                                                                   
 가로와 세로 크기의 값
 
 - wrap_content :  뷰에 들어 있는 내용물의 크기에 자동으로 맞춘다.
 - match_parent : 뷰에 담고 있는 뷰 그룹의 여유 공간을 꽉 채운다
 - 숫자로 크기기 지정 : 숫자를 사용해 크기를 지정한다. 크기는 dp나 px와 같은 단위가 꼭 있어야 한다.
