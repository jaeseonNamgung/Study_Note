# 뷰(View)
뷰는 레이아웃, 위젯이라고도 부르며 화면을 구성하는 인터페이스의 구성 요소이다. 
즉 뷰는 안드로이드 화면을 구성요소 이며 뷰는 화면에 보이는 것들이다. 

자주 사용하는 뷰
1. EditText : 사용자가 직접 텍스트를 입력할 수 있는 뷰
2. Button : 버튼을 터치 할 경우 버튼이 눌리는 효과를 보여주는 뷰 
3. TextView : 텍스트를 출력하는 뷰
4. ImageView : 이미지를 출력하는 뷰

이 4가지 기능은 모두 뷰(View)를 상속받는다. 즉 화면에 보이게 하는 기능을 가진 요소들은 받드시 뷰(View)를 상속받아야 한다. 
하지만 뷰는 화면 어디에 기능을 위치해야 되는지 알지 못한다. 이러한 화면에 기능을 위치하기 위해서는 뷰 그룹(ViewGroup)이 필요하다.

# 뷰 그룹(View Group)
뷰는 화면을 보이는 것들이라고 한다면 뷰 그룹은 화면에 어디에 위치해야하는지 정해주는 구성요소이다. 

자주 사용하는 뷰 그룹
1. ConstraintLayout : constraint는 제약이라는 뜻을 가지며 제약 조건을 통해 화면의 위치나 크기를 결정해주는 레이아웃이다.
1. LinearLayout : 박스(Box) 모델이다. 즉 한 쪽 방향으로 차례대로 뷰를 추가하는 레이아웃이다. 
2. RelativeLayout : 부모 뷰 또는 자식 뷰의 상대적 위치 관계를 정의하여 UI를 배치하는 레이아웃이다.
4. ### 레이아웃(Rayout)
안드로이드의 레이아웃이란 사용자에게 어떻게 효과적으로 보여줄 것인지 고려해서 UI를 만들고, 위젯들을 구성하고 배치하여 호율적이며 조금 더 편하게 사용할 수 있게 해주는 기능이다.
